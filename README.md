## 核心结论：升级方案，IP 不变

搬瓦工升级套餐的机制是**原地扩容**，不是重新开一台机器。整个过程在同一台物理节点上完成，IP 地址、已有数据、系统配置全部保留，不需要重建任何东西。

## 但有一种情况会让 IP 变——主动切机房

搬瓦工升级方案本身不改变 IP，这点没争议。但有个地方容易搞混：**切机房会改变 IP**。

搬瓦工的很多套餐支持在 KiwiVM 后台一键切换机房，比如 CN2 GIA-E 套餐可以在十几个机房之间迁移。每次切到新机房，IP 都是那个机房随机分配的新地址，和之前的不一样。

这两件事不同——升级套餐和切机房是两个独立操作。

升级套餐时，你可以同时选新的计费周期，但机房不会被强制换掉。如果你在升级过程中主动选了一个不同的机房，那自然会得到新 IP。但如果你只是升配置不动机房，IP 就稳稳的不动。

还有一个细节：如果你当前机房的 IP 已经被封了，切回同一机房有时候还是能拿回原来的 IP（前提是这个 IP 没被其他人占走）。不过这种情况比较边缘，别太依赖。

顺便说一个很多人搜升级方案时其实在担心的问题：**IP 被封怎么办**。

搬瓦工已经取消了免费更换 IP 服务，现在只有付费换 IP 一条路，费用大概是 $8.79/次，申请之后最多 24 小时内处理完。流程是在账户后台找到 IP Change 页面，找到对应 VPS 点击 Request IP Change，付款就行。

有一点要先做的：确认 IP 真的被封了，而不是本地网络、SSH 端口或者 VPS 没开机的问题。国内 ping 不通但国外能通，基本确认是被封了。如果只是 SSH 连不上，先排查端口和防火墙，别急着花钱换 IP。

如果你的套餐是月付而且 IP 刚被封不久，算一下剩余价值，有时候重新买一台新 VPS 比付费换 IP 更划算。年付套餐就老老实实付 $8.79 换一个。

## 常见问题

**Q：升级套餐之后，我的域名解析需要重新改吗？**

不用。IP 没变，域名解析完全不受影响，升级完可以直接正常访问。

**Q：从 CN2 GT 套餐升级到 CN2 GIA-E 套餐，IP 会变吗？**

这个要分情况。如果你的当前套餐支持直接在后台升级到 CN2 GIA-E，且选择原机房，IP 通常保持不变。但如果系统要求你迁移到新机房才能使用 CN2 GIA-E 线路，那么机房变了，IP 自然会换。具体看后台升级页面显示的可选项，切换机房前会有提示。

**Q：升级之后 VPS 会断线多久？**

升级后会触发一次 VPS 重启，断线时间很短，通常几分钟内就起来了。如果跑着关键服务，建议选业务低峰期操作。

**Q：搬瓦工升级套餐是立刻生效还是等下次续费？**

立刻生效。系统会计算当前套餐剩余价值，你只付差价，升级后新配置马上到位。

**Q：升级后 IP 没问题，但服务器配置好像没变，怎么回事？**

登录 KiwiVM 后台手动重启一次，再用 `free -m` 确认内存。有时候系统重启没有自动触发，手动操作一下就好了。
